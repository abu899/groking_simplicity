# Part 2

## Ch 10 

### 코드의 냄새: 함수 이름에 있는 암묵적 인자

문제점([암묵적 인자 V0](../src/main/kotlin/com/study/groking/part2/ch10/ImplicitArgumentV0.kt))
- 함수 이름에 있는 일부가 인자처럼 동작하는 것
- 값을 명시적으로 전달하지 않고 함수 이름의 일부로 전달
- 중복의 문제

해결 방안 : 암묵적 인자를 드러내기 리팩터링을 사용 ([암묵적 인자 V1](../src/main/kotlin/com/study/groking/part2/ch10/ImplicitArgumentV1.kt))

1. 함수 이름에 있는 암묵적 인자를 확인
2. 명시적인 인자로 추가
3. 하드코딩 된 값을 새로운 인자로 변경
4. 함수를 호출하는 곳을 변경

### 일급 함수인 것과 아닌 것을 구별하기

일급 함수로 할 수 있는 것

- 변수에 할당
- 함수의 인자로 넘기기
- 함수의 리턴값으로 받기
- 배열이나 객체에 담기
